// Getting Started

Prerequisites

Before running Forensicator, make sure the target machine meets these requirements:

  • Windows 7 / Server 2008 R2 or later
  • PowerShell 5.1 or later
  • Administrator privileges (mandatory for full artifact access)
  • Optional — WinPmem in the Forensicator-Share directory for RAM capture
  • Optional — etl2pcapng in the Forensicator-Share directory for PCAP conversion
  • Optional — SigmaRules in the Forensicator-Share directory for Malicious Activity Detection
  • Optional — SQLLite3 in the Forensicator-Share directory for Browsing History Extraction
⚠️
PowerShell execution policy must allow script execution. Run Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass in your elevated session if needed.
  • Any major Linux distribution (Debian, Ubuntu, RHEL, CentOS, Arch, Alpine tested)
  • Bash 4.0 or later
  • Root privileges (run with sudo)
  • Only standard OS tools used — no external dependencies required
💡
Forensicator deliberately avoids net-tools and other deprecated packages so it works across all modern distributions out of the box.
  • macOS 10.15 Catalina or later
  • Bash or Zsh (default shell on modern macOS)
  • Root / sudo privileges
  • Full Disk Access granted to Terminal in System Settings → Privacy & Security
⚠️
macOS SIP (System Integrity Protection) may restrict access to some system paths. Disabling SIP is not recommended in a production environment — Forensicator will collect what's accessible.

Installation

Forensicator requires no installation. Clone or download the repository to the target system or a trusted USB drive.

Option 1 — Git Clone (Recommended)

Bash / PowerShell
git clone https://github.com/Johnng007/Live-Forensicator.git
cd Live-Forensicator

Option 2 — Direct Download

Download the latest release ZIP from the Releases page, extract it, and navigate to the appropriate platform folder.

Folder Structure
Live-Forensicator/
├── Windows/
│   ├── Forensicator.ps1           # Main PowerShell script
│   ├── Forensicator-Share/        # Contains optional tools and resources
│   └── README.md
│   └── config.json                # Forensicator's configuration file
│   └── forensicator-runtime.js    # Forensicator's dashboarding runtime file
│   └── version.txt                # Forensicator's version file
├── Linux/
│   ├── Forensicator.sh            # Main Bash script
│   └── README.md
└── macOS/
    ├── Forensicator.sh            # Main Shell script
    └── README.md
💡
For live incident response, copy the relevant platform folder to a write-protected USB drive so you can deploy it on any machine without internet access.

Windows — Basic Run

Navigate to the Windows/ directory and run the script with Administrator privileges.

1
Open an elevated PowerShell window
Right-click the Start menu → Windows PowerShell (Admin) or Terminal (Admin).
2
Navigate to the Windows folder
PowerShell
cd C:\Path\To\Live-Forensicator\Windows
3
Run the script
PowerShell
.\Forensicator.ps1
Without flags, Forensicator runs a standard collection covering system information, user accounts, network state, processes, startup items, scheduled tasks and malicious activity check.
4
Review the output
A folder named after the computer name (e.g., DESKTOP-J7KX2\) is created in the same directory. Open index.html inside it with any browser.
// Windows

Windows — All Flags

Forensicator flags are additive — combine as many as your investigation requires. Each flag activates one additional collection or analysis module.

Flag Value Description
-VERSION VERSION Display the current version of Forensicator.
-UPDATE UPDATE Check for updates and updates to the latest version of Forensicator.
-EVTX EVTX Export windows event logs to EVTX files. This can be useful if you want to analyze the logs with external tools or preserve them for future reference.
-RAM RAM Acquire a full physical memory dump using WinPmem. Requires winpmem_mini_x64_rc2.exe or winpmem_mini_x86_rc2.exe to be present in the script directory.
-PCAP PCAP Capture 120 seconds of live network traffic using netsh trace, then convert it to a standard PCAP file via etl2pcapng for Wireshark analysis.
-WEBLOGS WEBLOGS Collect IIS web server access logs so you can scan or review them for exploitation patterns including directory traversal, SQL injection attempts, and web shell access.
-LOG4J LOG4J Scan application logs and JAR files for indicators of Log4Shell (CVE-2021-44228) exploitation attempts including JNDI lookup strings.
-RANSOMWARE RANSOMWARE Run a scan for ransomware indicators and suspicious file activity and name patterns.
-HASHCHECK HASHCHECK Run a hash check against known malicious file hashes to identify potential threats.
-ENCRYPTED ENCRYPTED AES-encrypt the entire artifact output folder after collection completes. You will be prompted for a passphrase. Useful for secure chain-of-custody transport.
-DECRYPTED DECRYPTED AES-decrypt a copy of an encrypted artifact file. You will be prompted for a passphrase.
-OPERATOR "Your Name" Embed the analyst's name in the report header for documentation purposes. Use quotes for names with spaces.
-CASE "ID" Embed a case or incident reference number in the report header (e.g., IR-2026-001).
-TITLE "Title" Set a custom title for the HTML report — useful for distinguishing reports across multiple machines in a multi-host investigation.
-LOCATION "LOCATION" Set the physical location of the investigation — useful for legal and regulatory purposes.
-DEVICE "DEVICE" Set the device name or identifier for the investigation — useful for distinguishing reports across multiple machines in a multi-host investigation.
ℹ️
Most flags accept the same value as their name (e.g., -EVTX EVTX, -RAM RAM). This is intentional — the flag name acts as a toggle.

Windows — Examples

Quickstart (standard collection)

PowerShell
.\Forensicator.ps1

Full investigation with RAM, PCAP, and Event Logs

PowerShell
.\Forensicator.ps1 -EVTX EVTX -RAM RAM -PCAP PCAP

Ransomware & Hashcheck

PowerShell
.\Forensicator.ps1 -RANSOMEWARE RANSOMWARE -HASHCHECK HASHCHECK

Web server compromise

PowerShell
.\Forensicator.ps1 -WEBLOGS WEBLOGS -LOG4J LOG4J

Encrypted collection for legal/case handoff

PowerShell
.\Forensicator.ps1 -ENCRYPTED ENCRYPTED

Maximum deep-dive (all modules)

PowerShell
.\Forensicator.ps1 -EVTX EVTX -RAM RAM -PCAP PCAP -RANSOMWARE RANSOMWARE -HASHCHECK HASHCHECK -WEBLOGS WEBLOGS -LOG4J LOG4J
⚠️
The -RAM flag pauses execution for the duration of the memory dump (size depends on installed RAM). The -PCAP flag pauses for 120 seconds while capturing traffic. Plan accordingly for time-sensitive engagements.
// Windows — New in v4.1.1

Sigma Rule Support

Sigma is the industry-standard, SIEM-agnostic detection language. Forensicator v4.1.1 bundles a curated Sigma rule set and evaluates machine's eventlogs against them automatically.

How it works

1
Ran by default
When .\forensicator.ps1 is run with or without flags.
2
Rules are evaluated
Each bundled Sigma rules are parsed and its conditions are matched against the windows event logs in realtime.
3
Matches are reported
Triggered rules appear in the HTML report under the Sigma Rules and also the Detections section with rule name, severity, MITRE ATT&CK technique, and the matching log entry.

Usage

PowerShell
# Sigma requires EVTX collection to have data to evaluate against
.\Forensicator.ps1

Custom Configurations

PowerShell
# The configuration file allows customization of Sigma rule evaluation
config.json

  "sigma": {
    "days_back": 7,               # Set this to how far back in days to evaluate logs
    "min_level": "medium",        # Minimum severity level to report (low, medium, high)
    "enable_sysmon": false,       # Set to true if you have Sysmon logs and want to include them in Sigma evaluation
    "include_source_ids": [],     # List of Event Log source IDs to include (e.g., ["sysmon_process_creation"]) default is all
    "exclude_source_ids": [],     # List of Event Log source IDs to exclude default is none
    "include_log_names": [],      # List of Event Log names to include (e.g., ["Microsoft-Windows-Sysmon/Operational"]) default is all
    "exclude_log_names": [],      # List of Event Log names to exclude default is none
    "include_categories": [],     # List of Event Log categories to include (e.g., ["process_creation"]) default is all
    "exclude_categories": []      # List of Event Log categories to exclude default is none
  }

Adding custom rules

We are working on an easier way to allow for custom .JSON rule files in the Forensicator-Share\rules\ directory so Forensicator will automatically pick them up.

💡
The SigmaHQ repository contains thousands of community-maintained rules. We actively collect these rules and convert them to the way Forensicator can parse them cleanly.
The Sigma runs can take several days to complete so by default we have set in the config.json file to skip sysmon logs and limit the days back to 7days. You are free to enable sysmon logs and adjust the days back as needed.

// Linux

Linux — Flags

Flag Value Description
-operator "Name" Analyst name to embed in the report header.
-case "ID" Incident or case reference number for documentation.
-title "Title" Custom title for the HTML report.

The Linux module runs a comprehensive collection by default, covering:

  • System information (kernel, uptime, hardware)
  • User accounts, groups, sudoers, and login history
  • Running processes and open files (lsof)
  • Network interfaces, active connections, and listening ports
  • Scheduled jobs (cron, at, systemd timers)
  • Installed packages and recently modified files
  • Syslog, auth log, and bash history artifacts
  • Malicious executable pattern detection
  • Suspicious shell command history scanning

The HTML Report

The primary output is a self-contained index.html report — no server needed, just open it in any browser.

📋
Summary Dashboard
Report header with case metadata, machine details, collection timestamp, and analyst name, detection summary.
🔍
Searchable Tables
Every collection result is presented in a filterable, sortable table. Search across all sections instantly.
⚡
Sigma Detections
Rule matches highlighted with severity level, MITRE technique ID, and the triggering log entry.
🦠
Hash Hits
Executables matching malicious hash databases are flagged in red with the malware family name if available.

IOC Browser Hits

Export Options

Artifact Folder Structure

Alongside the HTML report, Forensicator saves raw artifact files in the same output folder for deeper analysis.

Output Folder (Windows example)
DESKTOP-J7KX2\
├── index.html                          # Main report
├── forensicator-runtime.js             # The dashboard rendering helper file
├── SigmaFindings.csv                   # Sigma rule matches as a csv file - Helpful for further analysis
├── RAM\                                # Contains RAM dump (if -RAM)
├── PCAP\                               # Network capture (if -PCAP)
├── HashMatches\                        # Contains hash matches (if -HASHCHECK)
├── GroupPolicy\                        # Contains Group Policy artifacts (if Joined to a Domain)
├── EVTX\                               # Contains event log files (if -EVTX)
├── IISLogs\                            # IIS logs (if -IIS)
├── TomCatLogs\                         # Tomcat logs (if -TOMCAT)
└── LOG4J\                              # Log4j logs (if -LOG4J)
// Output

Artifact Encryption

Use the -ENCRYPTED ENCRYPTED flag to AES-encrypt the entire output folder before transport. This maintains chain of custody and protects sensitive data.

PowerShell
.\Forensicator.ps1 -ENCRYPTED ENCRYPTED

You will be prompted to enter and confirm a passphrase. The output folder is then first zipped before being encrypted into a single .forensicator file.

PowerShell (decrypt)
# Decryption is possible with the same copy of forensicator.ps1
.\forensicator.ps1 -DECRYPTED DECRYPTED ".\DESKTOP-J7KX2.forensicator"
🔐
Store your passphrase securely. The artifacts cannot be recovered without it. For legal proceedings, document the passphrase using your organisation's evidence handling procedures.
// Advanced

Unattended / Automated Mode

For mass deployment or SOAR integration, Forensicator can run fully unattended by providing all parameters at invocation time.

PowerShell — Example deployment script
# Deploy via PSExec or RMM to a remote host
$machines = @("SRV-WEB-01", "SRV-DB-02", "WKSTN-055")

foreach ($machine in $machines) {
    Invoke-Command -ComputerName $machine -ScriptBlock {
        .\Forensicator.ps1 -EVTX EVTX `
          -OPERATOR "SOC-Automation" -CASE "IR-2026-SWEEP" -LOCATION "Data Center" -DEVICE $machine -TITLE "Investigation Sweep"
    }
}
ℹ️
Avoid using -RAM and -PCAP in unattended sweeps — they block execution for several minutes per host. Use them for targeted deep-dives on specific machines of interest.
// Logging

Script Execution Logging

All Forensicator activities are logged in a very rich way, logs can be found inside LOGS\ folder in the root of theWindows\ it contains detailed information about each module's execution, errors, and other relevant events.

CMD / PowerShell
Log files are named like below
HostName_YY-MM-DD_HH-MM-SS.log                Contains raw log of the terminal execution output
HostName_YY-MM-DD_HH-MM-SS_findings_only.csv  Contains a csv of all findings only - Useful for SIEM ingestion
HostName_YY-MM-DD_HH-MM-SS_structured.csv     Contains CSV output of all terminal output (including findings) - Useful for SIEM ingestion
HostName_YY-MM-DD_HH-MM-SS_structured.json    Contains structured JSON output of all findings - Useful for SIEM ingestion

The logging system is designed to provide comprehensive insights into the script's execution, making it easier to troubleshoot issues and analyze results.

⚠️
The csv and JSON log files are designed to be ingested into SIEM systems for further analysis.
// IOC

Indicators of Compromise (IOC) Files

Forensicator during execution will attempt to fetch IOC Files (e.g., malware hashes, malicious IPs/URLs) from threat intel repos abuse.ch and drop them in the Forensicator-Share\ these are then used to match hashes and browsing history URLs.

PowerShell
# The configuration file allows customization of Sigma rule evaluation
config.json

  "hash_source": [
    "https://bazaar.abuse.ch/export/txt/md5/recent/"
  ],
    "url_source": [
    "https://urlhaus.abuse.ch/downloads/text_recent/"
  ],

You can change these sources in the config.json file to point to different threat intelligence feeds.

⚠️
The downloaded IOC files are text files with data separated by newlines.

Tips & Tricks

Run from a USB

If you wish to further minimize the risk of contaminating your forensic environment, you can run Forensicator from a USB drive.

Speeding up repeated runs

During active engagements where you run Forensicator multiple times, you may skip -RAM and -PCAP on repeat runs — they can be time-consuming and the data is unlikely to change significantly between runs. You can also set net_capture_duration in the config.json file — the default is 120s.

Parsing the output programmatically

All CSV artifact files in the output folder follow a consistent column structure. You can ingest them directly into Excel, Splunk, Elastic, or a Python/PowerShell pipeline for further correlation.

All CSV and JSON log files in the LOGS\ folder follow a consistent column structure. You can ingest them directly into Excel, Splunk, Elastic, or a Python/PowerShell pipeline for further correlation.

Multi-host correlation

Use the -CASE flag with the same case ID across all machines in an investigation. This makes it easy to correlate timestamps and findings across multiple HTML reports.

Keeping IOC's and Rules up to date

The IOC files automatically update on execution with the latest data if the files are missing in the Forensicator-Share folder or if the files are out of date. For Sigma rules, you can manually update them using the following commands:

Bash / PowerShell — update rule pack
# Pull latest rules from SigmaHQ
git clone  https://github.com/Johnng007/Live-Forensicator 

# Copy Windows rules
cp -r New_Clone\Live-Forensicator\Windows\Forensicator-Share\rules\windows\sigma\* Current_Copy\Live-Forensicator\Windows\Forensicator-Share\rules\windows\sigma
